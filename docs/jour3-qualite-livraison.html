Jour 3 - Atelier synthèse en binôme (projet noté)

Objectif du jour : réaliser un mini-projet API en binôme à partir d'une expression des besoins, puis faire une démonstration notée.

Format : atelier concentré sur une après-midi.

1. Sujet atelier : Challenge Arena

Expression des besoins

Contexte : une association étudiante veut organiser des défis techniques entre participants et afficher un classement public.

Besoin métier : disposer d'une API simple pour créer des participants, créer des défis et comptabiliser les points.

Fonctionnalités obligatoires (MVP)

  1. Créer un participant
  2. Créer un défi (titre, difficulté, points)
  3. Valider un défi pour un participant (ajout des points)
  4. Afficher le classement global

Contraintes techniques

  • API REST en JSON
  • Validation minimale des entrées
  • Gestion d'erreurs : 400, 404, 500
  • Stockage au choix : JSON ou SQLite
  • README d'exécution obligatoire

Critères d'acceptation

  • Le scénario complet est démontrable de bout en bout
  • Le classement reflète correctement les points
  • Les erreurs sont traitées proprement
  • Le projet peut être lancé par un tiers en quelques commandes

2. Répartition du travail à deux

RôleResponsabilités
Participant A Participants + classement + validations associées
Participant B Défis + attribution des points + validations associées
En commun Gestion d'erreurs, README, tests manuels, démo finale

4. Grille de notation

CritèrePoints
Respect de l'expression des besoins8

Validation et gestion des erreurs3
Démonstration fonctionnelle3
README et autonomie d'exécution2

Total : 20 points.

5. Checklist de rendu binôme

  • Les endpoints MVP fonctionnent.
  • Le classement est correct.
  • Les erreurs principales renvoient les bons statuts.
  • Le README explique installation, lancement et test.
  • La démo suit un scénario clair.
