Node.js Niveau I — Plan de cours détaillé (3 jours)

Formation : Initiation Node.js
Durée : 3 jours
Version cible : branche LTS Node.js 24 (état constaté au 30/04/2026)

Fil rouge gamifié : Campus Quest API

Les participants construisent un mini-jeu de progression :

  • Joueur (pseudo, niveau, XP)
  • Quêtes (facile, moyen, difficile)
  • Récompenses (points, badges)
  • Classement global

Ouverture : histoire et architecture de Node.js

Repères historiques

  • Naissance en 2009 pour proposer un serveur JavaScript orienté I/O non bloquantes.
  • Montée en puissance grâce à npm et à la culture open source.
  • Maturité actuelle : versions LTS, gouvernance stable, usage massif en backend et tooling.

Briques techniques à connaître dès le départ

  • V8 : moteur JavaScript.
  • libuv : event loop, I/O, timers, file system, réseau.
  • APIs natives Node : fs, http, path, stream, crypto.
  • Modules et runtime : CommonJS, ESM, chargement de dépendances.
  • npm : dépendances, scripts, distribution de packages.

JOUR 1 — Fondations Node.js et CLI gamifiée

Module 1 : Installation et prise en main

Objectifs : comprendre Node.js, installer l'environnement, lancer ses premières commandes.

  • Node.js runtime, npm, scripts
  • Structure d'un projet
  • Premier script exécutable

Module 2 : JavaScript serveur et modules

Objectifs : maîtriser les bases JavaScript nécessaires côté serveur.

  • ESM import/export
  • Modules natifs (node:fs/promises, node:path)
  • Organisation claire des fichiers

Module 3 : Asynchrone pas à pas

Objectifs : utiliser correctement Promises et async/await.

  • Lecture détaillée de l'event loop avec schéma pédagogique
  • Opérations I/O
  • try/catch et messages d'erreurs utiles
  • Écriture et lecture JSON

Module 4 : Défis CLI (gamification)

Objectifs : livrer une première version jouable en terminal.

  • Commande create-player
  • Commande accept-quest
  • Commande complete-quest
  • Commande show-stats

Défi : système de points (bronze, argent, or) selon la qualité du code et la gestion des erreurs.

Livrable fin J1 : CLI Campus Quest fonctionnelle avec stockage JSON.

JOUR 2 — API REST gamifiée avec Express

Module 5 : Bases HTTP et REST

Objectifs : poser les conventions API pour le jeu.

  • Ressources : /players, /quests, /scores
  • Codes HTTP cohérents
  • Contrat JSON commun

Module 6 : Express pas à pas

Objectifs : construire les routes principales.

  • GET /players
  • POST /players
  • POST /players/:id/quests/:questId/complete
  • GET /leaderboard

Module 7 : Qualité d'API

Objectifs : validation, gestion des erreurs, structure propre.

  • Validation des entrées
  • Middleware d'erreur
  • Séparation routes/controllers/services

Module 8 : Challenges API (gamification)

Objectifs : stabiliser l'API et valider les flux bout en bout.

  • Challenge 1 : endpoint bonus /daily-quest
  • Challenge 2 : endpoint /top-3
  • Challenge 3 : anti-triche (pas de gain XP en double)

Livrable fin J2 : API Campus Quest complète, testée manuellement.

JOUR 3 — Atelier synthèse binôme (après-midi) noté

Module 9 : Cadrage de l'atelier

Objectifs : lire une expression des besoins et définir un plan d'implémentation.

  • Lecture du besoin
  • Choix du MVP
  • Répartition des rôles à 2

Expression des besoins (atelier)

Contexte : une association étudiante veut une mini-API pour organiser des défis entre participants.

Nom du projet : Challenge Arena.

Fonctionnalités attendues (MVP) :

  1. Créer un participant
  2. Créer un défi (titre, difficulté, points)
  3. Valider un défi pour un participant (ajout de points)
  4. Afficher le classement des participants

Contraintes :

  • API REST en JSON
  • Validation minimale des entrées
  • Gestion des erreurs (400, 404, 500)
  • Stockage libre : JSON ou SQLite

Critères d'acceptation :

  • Les 4 fonctionnalités sont démontrées en direct
  • Le code est structuré
  • Un README permet d'exécuter le projet rapidement
