Jour 2 - API REST Campus Quest avec Express

Objectif du jour : transformer la version CLI en API REST propre et testable.

Livrable fil rouge : API Campus Quest avec routes joueurs, quêtes et classement.

1. Rappels HTTP essentiels

Méthodes et usages

  • GET : lire des données
  • POST : créer une ressource
  • PATCH : modifier partiellement
  • DELETE : supprimer

Codes de réponse à utiliser

  • 200 : succès lecture/modification
  • 201 : ressource créée
  • 400 : données invalides
  • 404 : ressource non trouvée
  • 500 : erreur serveur

2. Serveur Node natif (sans framework)

Exemple simple - endpoint health

import http from "node:http";

const server = http.createServer((req, res) => {
  if (req.method === "GET" && req.url === "/health") {
    res.writeHead(200, { "content-type": "application/json" });
    res.end(JSON.stringify({ status: "ok" }));
    return;
  }

  res.writeHead(404, { "content-type": "application/json" });
  res.end(JSON.stringify({ error: "not found" }));
});

server.listen(3000);

Exemple technique - routing + body JSON

import http from "node:http";

const players = [];

const server = http.createServer((req, res) => {
  if (req.method === "GET" && req.url === "/players") {
    res.writeHead(200, { "content-type": "application/json" });
    res.end(JSON.stringify(players));
    return;
  }

  if (req.method === "POST" && req.url === "/players") {
    let body = "";
    req.on("data", chunk => { body += chunk; });
    req.on("end", () => {
      try {
        const data = JSON.parse(body || "{}");
        if (!data.pseudo || data.pseudo.trim().length < 3) {
          res.writeHead(400, { "content-type": "application/json" });
          res.end(JSON.stringify({ error: "pseudo invalide" }));
          return;
        }
        players.push({ id: players.length + 1, pseudo: data.pseudo.trim() });
        res.writeHead(201, { "content-type": "application/json" });
        res.end(JSON.stringify(players[players.length - 1]));
      } catch {
        res.writeHead(400, { "content-type": "application/json" });
        res.end(JSON.stringify({ error: "json invalide" }));
      }
    });
    return;
  }

  res.writeHead(404, { "content-type": "application/json" });
  res.end(JSON.stringify({ error: "not found" }));
});

server.listen(3000);

Pourquoi montrer ça avant Express ? Pour comprendre ce qu'Express simplifie : le routing, le parsing du body, la gestion des erreurs et la lisibilité globale.

3. Passer à Express

Étape 1 - Installation

npm install express

Étape 2 - Serveur minimal

import express from "express";

const app = express();
app.use(express.json());

app.get("/health", (_req, res) => {
  res.status(200).json({ status: "ok" });
});

app.listen(3000);

4. Exemples

Exemple 1 - Route POST avec validation

app.post("/players", (req, res) => {
  const pseudo = req.body?.pseudo?.trim();
  if (!pseudo || pseudo.length < 3) {
    return res.status(400).json({ code: "VALIDATION_ERROR", message: "pseudo invalide" });
  }
  return res.status(201).json({ id: 1, pseudo });
});

Exemple 2 - Route métier avec service

app.post("/players/:id/quests/:questId/complete", async (req, res, next) => {
  try {
    const result = await questService.complete(req.params.id, req.params.questId);
    return res.status(200).json(result);
  } catch (err) {
    return next(err);
  }
});

Exemple 3 - Middleware d'erreur unifié

app.use((err, _req, res, _next) => {
  const status = err.statusCode || 500;
  res.status(status).json({
    code: err.code || "INTERNAL_ERROR",
    message: err.message || "erreur interne",
    details: err.details || []
  });
});

5. Ateliers et exercices pratiques

Atelier 1 - Contrat API

Définir précisément 5 endpoints :

  1. Méthode HTTP
  2. URL
  3. Payload d'entrée (valide + invalide)
  4. Réponse de succès
  5. Réponse d'erreur

Atelier 2 - Implémentation guidée

Implémenter les routes suivantes avec validations minimales :

  • POST /players
  • POST /quests
  • POST /players/:id/quests/:questId/complete

Contraintes : statut correct + message explicite + refus doublon de validation.

  1. Limiter le nombre de requêtes
  2. Ajouter helmet
  3. Configurer un CORS explicite
  4. Masquer les traces internes en réponse

Atelier 6 - Qualité d'exploitation

  1. Créer un endpoint /health détaillé (status, uptime, version)
  2. Ajouter des logs structurés (méthode, route, status, durée)
  3. Créer une collection API prête pour la démo

Exercice bonus

Ajouter un endpoint GET /players/:id/history qui renvoie les quêtes réalisées par un joueur.

Hard mode : ajouter GET /players/:id/progression (xp, niveau, progression, 3 dernières quêtes).
