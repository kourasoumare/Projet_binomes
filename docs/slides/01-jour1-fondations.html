Macrotasks : setTimeout (même 0), setInterval, I/O callbacks, setImmediate

Durée variable : ce n'est pas une horloge fixe.

Exemple 1 - Lecture JSON

import { readFile } from "node:fs/promises";

const playersPath = new URL("../data/players.json", import.meta.url);
const raw = await readFile(playersPath, "utf8");
const players = JSON.parse(raw);
console.log(players.length);

node: = module natif Node.js, pour éviter la confusion avec un package npm.

Exemple 2 - Fichier + moyenne

const csv = await readFile("./data/scores.csv", "utf8");
const lines = csv.trim().split("\n").slice(1);
const scores = lines.map(l => Number(l.split(";")[1]));
const avg = scores.reduce((a,n)=>a+n,0) / scores.length;
console.log(avg.toFixed(2));

Exemple 3 - Gestion d'erreur fichier

async function readJsonSafe(path) {
  try {
    const raw = await readFile(path, "utf8");
    return JSON.parse(raw);
  } catch (err) {
    if (err.code === "ENOENT") return [];
    throw err;
  }
}

C'est quoi Campus Quest ?

Un jeu d'énigmes et de résolution de problèmes, exposé via une API JSON :

  1. Recevoir une énigme
  2. Proposer une réponse
  3. Résoudre un problème pour gagner de l'XP
  4. Suivre la progression et le classement

Dans sandbox-server, chaque action passe par une route API (URL + méthode HTTP + JSON).

Exemple : POST /enigmes/:id/answer valide une tentative de réponse.

Livrable J1 : une API JSON exécutable de bout en bout autour du jeu Campus Quest.

Qui fait quoi ? (flux réel)

POST /players avec { "name": "Alice" }

  1. server.correction.js reçoit la requête HTTP
  2. Valider la règle métier (réponse correcte ? déjà résolue ?)
  3. json-store.js lit/écrit les JSON
  4. Le serveur renvoie une réponse JSON

server.js : exemple minimal

app.post("/players", (req, res) => {
  const { name } = req.body;
  if (!name) return res.status(400).json({ error: "name is required" });
  res.status(201).json({ id: "p1", name, xp: 0, solvedEnigmes: [] });
});

Idée clé : lire les données de la requête, appliquer la validation, renvoyer un JSON.

Même logique métier, déclenchée par HTTP avec des réponses JSON cohérentes.

Lire / écrire un JSON

import { readFile, writeFile } from "node:fs/promises";

async function loadJson(path) {
  try { return JSON.parse(await readFile(path, "utf8")); }
  catch (err) { if (err.code === "ENOENT") return []; throw err; }
}

async function saveJson(path, data) {
  await writeFile(path, JSON.stringify(data, null, 2) + "\n", "utf8");
}

Pattern: lire → modifier en mémoire → réécrire le fichier.

Exercice unique J1 (sans CLI)

Implémenter Campus Quest en REST API avec persistance dans data/ :

  1. POST /players
  2. GET /enigmes (sans answer)
  3. POST /enigmes/:id/answer
  4. GET /leaderboard
  5. GET /health

Validation + XP

  1. Comparer la réponse envoyée avec la bonne réponse de l'énigme
  2. Si correct + pas déjà résolue : ajouter enigme.xp au joueur
  3. Tracer chaque tentative dans data/attempts.json
  4. Ne jamais redonner de XP sur une même énigme pour le même joueur

Comparer en mode tolérant : trim() + toLowerCase().
